\section{Executive Summary: The Work Must Outlive the Chat}

Agents can already produce valuable code, research, plans, operations, and
decisions. Yet the dominant product container is still a chat session. When
that session closes, the next Agent often receives a transcript, a summary, or
an unverified memory instead of an exact statement of what the project has
become. The model may be capable while the work remains fragile.

Kungfu addresses this continuity gap. Its highest ordinary product interface
is the \textbf{Project Cut}: a verifiable project settlement that binds the
accepted source state, the perspective used to understand it, the causal work
that occurred, the decisions that admitted the result, unresolved risk, and
the coordinate from which authorized work may continue. A Project Cut does not
replace those authorities. It gives the next human or Agent one bounded answer
to the question: \emph{where can this project safely continue?}

\calloutbox{The product promise}{Give your agent verified context. Keep the
work when the chat ends.}

\subsection{One Product Decision, Two Adoption Paths}

Users should encounter Project Cut continuity before they need to understand
the runtime beneath it. Agent builders, however, need an immediate technical
path. Kungfu therefore separates two responsibilities:

\begin{enumerate}
  \item embed \texttt{libkungfu} as the reference runtime for durable Facts,
  Episodes, verification, recovery, and Project Cut continuity; and
  \item build a participant-owned Hub that retains the vendor's user
  experience, models, identity, accounts, policies, cloud, admission rules,
  and customer relationship while exchanging portable responsibility evidence
  through KFD.
\end{enumerate}

These paths should proceed in parallel. A vendor does not need to rebuild a
journal, content-addressed evidence store, recovery engine, and cross-language
fact layer before delivering continuity. Nor should embedding a runtime force
the vendor to surrender its product boundary to a central Kungfu service.

\calloutbox{Recommended Agent Builder strategy}{Build your Hub. Do not rebuild
the runtime. Embed libkungfu now, prove one source-pinned Project Cut
loop, and develop your KFD-compatible Hub boundary in parallel.}

\subsection{What This Paper Claims}

Kungfu provides a concrete architecture, open source implementation, and
first-party evidence for work that survives session and Agent replacement. It
does not claim that a transcript is a Project Cut, that delivery is admission,
that an Episode proves completion, or that authentication grants authority. It
does not claim external vendor adoption, stable KFD Hub interoperability,
certification, or broad production readiness. The public interfaces described
here are alpha surfaces and should be adopted through exact source and evidence
coordinates.
