\section{Ecosystem}

Kungfu's open-source repository can be valuable beyond the default product
because agent work management is a meta-capability. A developer can build
extensions for software work. A researcher can build publication workflows. An
operator can build infrastructure runbooks. A personal user can build life and
work organization profiles.

The ecosystem should be shaped by low-friction consistency:

\begin{itemize}[leftmargin=*]
  \item one coherent monorepo for the core product where splitting would create
  agent friction;
  \item package APIs for Python and Node where agents naturally integrate;
  \item GUI/TUI entrypoints for humans who need visibility without more
  cognitive load;
  \item KFD and Buildchain release evidence so users and agents can inspect
  what a release claims to provide;
  \item extension contracts that let new profiles add semantics without
  weakening the core fact model.
\end{itemize}

This ecosystem does not require every user to adopt the initial private
dogfood semantics. It requires Kungfu to make it practical for users to define
their own durable work semantics and still participate in the same trust model.
